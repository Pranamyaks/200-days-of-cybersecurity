# Day 11 - Web, Network & Security Basics

##  HTTP vs HTTPS

### HTTP (HyperText Transfer Protocol)
- Not secure  
- Data sent in plain text  

### HTTPS (Secure HTTP)
- Encrypted using SSL/TLS   
- Secure communication  

##  Ports & Protocols

- Port = entry point for network communication  

### Common Ports:
- 80 → HTTP  
- 443 → HTTPS  
- 21 → FTP  
- 22 → SSH  

##  Firewall

Firewall is a security system that monitors and controls network traffic.

- Blocks unauthorized access  
- Allows safe traffic  

##  NAT (Network Address Translation)

NAT converts private IP addresses into a public IP.

### Example:
Many devices → One public IP (home WiFi)

##  Subnetting (Simplified)

Subnetting divides a network into smaller parts.

### Purpose:
- Improves performance  
- Enhances security  

##  How Internet Works

1. User enters URL  
2. DNS finds IP address  
3. Request sent to server  
4. Server responds  
5. Browser displays webpage  

##  Key Point
Internet works through multiple layers, protocols, and security mechanisms.

##  Cybersecurity Relevance
- HTTPS ensures secure communication  
- Firewall protects network  
- NAT hides internal network  
- Ports are used in attacks (port scanning)
